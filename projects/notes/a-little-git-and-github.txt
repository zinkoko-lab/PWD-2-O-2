一人で開発するために最低限覚えておけば十分な Git コマンドだけ に絞って、超わかりやすくまとめます。
（チーム開発で必要な複雑なコマンドは一旦全部無視してOK！）

✅ 一人で使う Git — 絶対に必要な8つだけ

⸻

## ① Git 初期化

最初に一度だけやる。

git init

※ 既に Git 管理されているフォルダでは不要。

⸻

## ② 状態を確認（必ずよく使う）

今の状況を知るコマンド。

git status

	•	どのファイルが変更されたか？
	•	どれがコミット対象か？
	•	どのブランチにいるか？

全部ここで確認できます。

⸻

## ③ ファイルを追加（ステージング）

Git に「このファイルをコミットします！」と教える操作。

git add ファイル名

すべて追加したい場合：

git add .


⸻

## ④ コミット（作業を保存）

メッセージは短くてOK。

git commit -m "メッセージ"

例：

git commit -m "add header component"


⸻

## ⑤ GitHub と接続（最初の1回だけ）

git remote add origin https://github.com/ユーザー名/リポジトリ名.git


⸻

## ⑥ main ブランチに変更

（最近は master ではなく main が主流）

git branch -M main


⸻

## ⑦ GitHub にアップロード（Push）

git push -u origin main

2回目以降は短くなる：

git push


⸻

## ⑧ GitHub からダウンロード（Pull）

自分が別の PC で作業した時などに必要。

git pull


⸻

🎯 これだけ覚えておけば一人開発は余裕！

最低限セット：

目的	コマンド
作業開始前	git pull
何を変えたか確認	git status
変更を追加	git add .
保存（コミット）	git commit -m "xxx"
GitHub へ送信	git push


⸻

🧩 【実際の一連の流れ】

毎日の作業はこれだけ：

git pull
# 何か作業する
git add .
git commit -m "内容"
git push

本当にこれで完結します。

⸻

🧠 もしブランチを使いたくなったら…

一人開発ならブランチは必須じゃないけど、覚えたいならこれだけ：

git branch feature/login
git checkout feature/login

この2つだけで十分。
